# Face-expression-recognition-system-based-on-OpenCV
Facial Emotion Recognition using CNN, Using OpenCV’s haar cascade xml we are getting the bounding box of the faces in the webcam. Then feed these boxes to the trained model for classification.
# 表情识别系统 - 功能说明文档

## 一、系统概述
本程序是一款基于 `tkinter` 搭建可视化界面、结合 OpenCV 人脸检测与 TensorFlow/Keras 深度学习模型的**实时表情识别系统**。程序可调用电脑摄像头，实时捕捉画面中的人脸并识别其表情类别，同时展示各类表情的概率占比，最终通过图形化界面直观呈现识别结果。

## 二、核心功能
| 功能模块 | 详细说明 |
|----------|----------|
| 可视化GUI界面 | 基于tkinter构建美观的图形化界面，包含标题、摄像头画面显示区、识别结果展示区、退出按钮 |
| 实时摄像头采集 | 调用电脑本地摄像头（默认索引0），实时采集视频画面并进行预处理（镜像翻转+固定尺寸缩放） |
| 人脸检测 | 加载OpenCV官方人脸检测分类器（haarcascade_frontalface_default.xml），精准检测画面中的人脸并绘制蓝色矩形框标注 |
| 多类别表情识别 | 基于预训练的Keras模型（best_emotion_model.h5），识别7种核心表情：愤怒、厌恶、恐惧、开心、中性、悲伤、惊讶 |
| 概率量化展示 | 计算并展示每种表情的概率占比（保留2位小数），同时标注概率最高的“当前表情” |
| 优雅退出机制 | 点击“退出”按钮可释放摄像头资源并关闭GUI窗口，避免资源占用 |
| 跨环境兼容 | 兼容Python开发环境与打包后的可执行文件（dist目录），自动识别运行环境并适配文件路径 |

## 三、技术栈
- **界面框架**：tkinter（Python内置GUI库）
- **计算机视觉**：OpenCV（摄像头操作、人脸检测、画面预处理）
- **图像处理**：PIL（将OpenCV图像转换为tkinter可显示格式）
- **深度学习**：TensorFlow/Keras（加载预训练模型、表情预测）
- **数值计算**：NumPy（数据格式转换、张量处理）
- **系统适配**：sys/os（识别运行环境、文件路径处理）

## 四、运行环境要求
### 1. Python版本
推荐 Python 3.7~3.10（兼容TensorFlow/Keras主流版本）

### 2. 依赖包安装
```bash
pip install opencv-python pillow tensorflow numpy
```

## 五、文件依赖
程序运行前需确保以下文件与代码文件（可视化界面.py）同目录（打包后需放入dist文件夹）：
1. `best_emotion_model.h5`：预训练的表情识别Keras模型文件
2. `haarcascade_frontalface_default.xml`：OpenCV人脸检测分类器文件

## 六、运行步骤
1. 安装上述依赖包；
2. 将模型文件和人脸检测分类器文件放入代码同级目录；
3. 运行`可视化界面.py`：
   ```bash
   python 可视化界面.py
   ```
4. 程序启动后自动调用摄像头，界面实时显示：
   - 摄像头画面（带人脸标注框）；
   - 当前识别出的表情；
   - 7类表情的概率占比（百分比）；
5. 点击“退出”按钮关闭程序，释放摄像头资源。

## 七、功能细节说明
1. **画面预处理**：摄像头画面默认镜像翻转（符合人眼视觉习惯），并缩放到600×500固定尺寸显示；
2. **人脸检测阈值**：最小检测人脸尺寸为40×40像素，保证小尺寸人脸也能被识别；
3. **表情识别流程**：
   - 从检测到的人脸区域裁剪灰度图；
   - 缩放到48×48像素（模型输入尺寸）并归一化；
   - 模型预测输出7类表情概率；
   - 提取概率最大值作为“当前表情”；
4. **界面布局**：
   - 标题区：居中显示“实时表情识别”（黑体32号加粗）；
   - 画面区：左侧50,120位置显示600×500摄像头画面；
   - 结果区：右侧700,200位置显示识别结果（黑体16号加粗）；
   - 按钮区：底部居中显示“退出”按钮（红色背景，黑体18号加粗）；
5. **异常处理**：
   - 检测到模型文件/人脸分类器文件缺失时，打印提示信息并退出；
   - 摄像头无法打开时，提示检查占用程序或修改摄像头索引；
   - 摄像头画面读取失败时，自动重试（10ms间隔）。

## 八、注意事项
1. 运行前需确保摄像头未被其他程序（微信、Zoom、钉钉等）占用；
2. 若摄像头调用失败，可尝试将代码中`cv2.VideoCapture(0)`改为`cv2.VideoCapture(1)`（多摄像头设备）；
3. 模型文件需与训练时的输入格式、类别顺序一致（7类表情顺序：愤怒→厌恶→恐惧→开心→中性→悲伤→惊讶）；
4. 打包为可执行文件时，需将模型文件和人脸分类器文件一同放入dist目录；
5. 程序运行时需保证足够的计算资源，低配置设备可能出现画面轻微卡顿。
